## What's in the harness

The Go test code lives at `internal/benchmarks/coordstore/`:

| Component | Files | What it is |
|---|---|---|
| **Chaos engine** | `chaos_protocol.go`, `chaos_process.go`, `chaos_server.go`, `chaosrunner.go` + tests | Process-level kill / restart / durability-probe runner with a clean `ChaosController` interface decoupled from coord-store specifics. Reusable for any component that wants reliability proof under fault injection. Records `KillEvent`s (loss window, last-ack lag, recovery time). |
| **Soak runner** | `soakrunner.go` + tests | Phase A steady-state workload driver; writes `scorecard.json` + `timeseries.jsonl` per backend per phase. |
| **Workload** | `workload.go`, `workload_soak_test.go`, `lifecycle_plateau_test.go` | Realistic-lifecycle workload (ga-w08fz): main creates with low close rate (long-lived tasks), wisps with high churn (plateau-shaped), batched read/scan patterns. |
| **Triage** | `triage.go` + test | Phase C report: walks all per-phase artifacts, produces `triage.md` + `triage.json` cross-backend comparison. |
| **Recorder** | `recorder.go` + test | Timeseries sampler (HeapInuse, RSS, AllocDelta, op-level p99/throughput). |
| **Adapters** | `adapters/{authorcore,bbolt,couchdb,dolt,postgres,sqlite}/` | One per backend; trivial wrappers around `coordstore.StoreAdapter`. External adapters register only when their DSN/URL env vars are set. |
| **Top-level tests** | `suite_test.go` | `TestBenchmarkSoakPhaseA`, `TestBenchmarkSoakCalibrate`, `TestBenchmarkSoakPhaseB`, `TestBenchmarkSoakTriage`, and `TestBenchmarkSoakDolt` (legacy/baseline). |

## Gating env vars

| Var | Purpose |
|---|---|
| `COORDSTORE_SOAK=1` | **Required** for any soak test. Default off â†’ CI-safe. |
| `COORDSTORE_SQLITE_CGO=1` (+ `-tags sqlite_cgo`) | Enables the sqlite-cgo adapter. |
| `COORDSTORE_DOLT_DSN` | Enables the dolt adapter â€” must point at an **isolated** throwaway dolt (never the city's :28232). |
| `COORDSTORE_RESULTS_DIR` | Where scorecards/timeseries/triage go (required). |
| `COORDSTORE_SOAK_DURATION` | Phase A length (default `2h`). |

## Launchers (this directory)

- `launch-dolt-baseline.sh` â€” `TestBenchmarkSoakDolt`. Needs `COORDSTORE_DOLT_DSN` pre-set.
- `setup-isolated-dolt.sh` â€” spins up a throwaway `dolt sql-server` on a non-:28232 port for the dolt baseline.

Each launcher writes a `launch.txt` provenance file with branch SHA + env into the results dir. Launchers derive Go paths from `go env` by default; override `GOPATH`, `GOCACHE`, `GOMODCACHE`, or `GOROOT` in the environment for local custom toolchains.

## Reading results

Per backend, per phase:
- `scorecard.json` â€” totals (ops, errors, duration, HeapInuse {Baseline,Peak,Steady}, RSS {Baseline,Peak,Steady}, AllocDelta) + per-op (p99, max, throughput, pass/fail vs aspirational target).
- `timeseries.jsonl` â€” periodic samples for plotting.

Cross-backend: `triage.md` + `triage.json` at the results-dir root (generated by `writeSoakTriage`).
