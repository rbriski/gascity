Per-class strangler flow & the two translation edges

Strangler: interface first, bd adapter (byte-identical), then a SQLite impl behind the same interface — clean drain-then-switch, one-line config flip per class.
Two edges only: the store emits a raw RowChanged (no upward dep); Layer 2/3 builds the typed domain event and the wire DTO. Layer 0 never knows about internal/api.
